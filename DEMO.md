# 演示流程

## 启动

1. 启动 Redis：`docker start redis-ticket`。
2. 启动后端：`cd backend && mvn spring-boot:run`。
3. 启动前端：`cd frontend && npm run dev`。
4. 浏览器访问：`http://localhost:5173`。

## 账号说明

系统包含后台角色和普通用户角色。后台角色仅允许在服务器本机通过 `localhost` 或 `127.0.0.1` 登录；局域网设备只用于普通用户购票演示。

## 录屏脚本

1. 管理员登录后台，查看运营概览。
2. 进入场馆管理，查看场馆、区域和座位图。
3. 进入座位模板，生成座位并查看 SVG 预览。
4. 进入场次管理，查看开售时间、售卖截止时间和场次状态。
5. 进入票档管理，查看票档、价格和后台库存配置。
6. 进入售票批次，初始化实时库存。
7. 普通用户浏览首页，进入演出详情。
8. 选择支持选座的场次，进入图形化选座页。
9. 选择座位和观演人，提交抢票。
10. 排队页显示系统正在确认库存和座位，不展示虚构前方人数。
11. 抢票成功后进入订单确认页。
12. 进入模拟支付页，完成支付。
13. 支付成功后进入我的票夹，查看电子票和入场码。
14. 检票员登录后台，输入票号完成核验。
15. 再次核验同一票号，展示重复入场拦截。
16. 普通用户在订单列表申请退票。
17. 管理员在退票审核页通过申请。
18. 查看库存池，确认取消、退票或售卖期外释放后的库存处理。
19. 查看站内信，确认抢票、出票、退票、核验通知。
20. 查看统计报表、操作日志和风控日志。

## 合规边界

系统只使用本地演示数据和管理员录入数据，不爬取真实票务平台，不接入真实支付、短信或邮箱。
